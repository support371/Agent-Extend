16. Eligibility Rule Engine (Business Logic)

16.1 Hard Blocks
	•	If seller not verified → cannot publish listing
	•	If buyer not verified → cannot start acquisition
	•	If species category restricted for destination → block checkout
	•	If required documents missing → block shipment booking

16.2 Soft Warnings
	•	Care level high + buyer readiness not acknowledged → warning + education prompt
	•	Route complexity high → auto-recommend priority handling

16.3 Region-Aware UI
	•	Destination selector updates:
	•	visible categories
	•	listing availability
	•	document checklist
